# Safe Ingestion Engine

Security-first ingestion API built with FastAPI, Celery, Redis, SQLAlchemy, and SSRF-safe HTTP collection.

## What this repo is

A clean modular monolith for safe web ingestion with:

- FastAPI API gateway
- Celery background jobs
- Redis-backed crawl budgets
- SSRF protection against private, loopback, link-local, and reserved networks
- robots.txt enforcement with configurable failure mode
- PII scrubbing by deterministic HMAC-based replacement
- SQLAlchemy persistence layer
- Structured JSON logging
- Docker and Kubernetes starter manifests
- Basic security regression tests

## Canonical entrypoints

- API app: `api.main:app`
- Celery app: `infrastructure.queue.celery_app:celery_app`
- Worker task: `infrastructure.queue.tasks.ingest_url_task`

## API

### `GET /health`
Returns service health.

### `POST /v1/ingest`
Header: `X-API-Key: <key>`

Body:
```json
{"url":"https://example.com"}
```

Response:
```json
{"job_id":"<celery-task-id>","status":"queued"}
```

### `GET /v1/jobs/{job_id}`
Returns job status and excerpt for the authenticated caller.

## Local development

1. Copy the environment template:
```bash
cp .env.example .env
```

2. Fill required values:
- `PII_SALT`
- `DASHBOARD_ADMIN_PASSWORD`
- `API_KEY_HASHES_JSON`

3. Start locally:
```bash
docker compose up --build
```

4. Open docs:
- `http://localhost:8000/docs`

## Security notes

This repo is production-oriented, but for serious production volume you should still:

- switch from SQLite to Postgres
- move secrets to a proper secrets manager
- add richer observability and alerting
- expand PII patterns for your compliance jurisdiction
- add more negative-path and concurrency tests
